# 工程可行性报告AI生成系统 - Agent架构方案

**版本**：1.0  
**日期**：2026-02-25  
**目标**：构建一个完整的Multi-Agent系统，自动生成专业的工程可行性报告

---

## 一、报告结构分析

根据国家发改委《投资项目可行性研究报告编写大纲（2023年版）》，一份完整的可行性报告包含以下章节：

### 政府投资项目通用大纲（11章）

| 章节 | 内容领域 | 核心要点 |
|------|----------|----------|
| 一、概述 | 项目基本情况 | 项目概况、单位概况、编制依据、结论 |
| 二、项目建设背景和必要性 | 宏观分析 | 背景、规划符合性、必要性 |
| 三、项目需求分析与产出方案 | 需求分析 | 需求分析、建设内容、产出方案 |
| 四、项目选址与要素保障 | 选址条件 | 选址、建設条件、要素保障（土地、资源） |
| 五、项目建设方案 | 技术方案 | 技术、设备、工程、数字化方案 |
| 六、项目运营方案 | 运营管理 | 运营模式、组织、安全、绩效 |
| 七、项目投融资与财务方案 | 财务分析 | 投资估算、盈利、融资、债务、财务可持续 |
| 八、项目影响效果分析 | 效果评估 | 经济、社会、生态环境、资源能源、碳排放 |
| 九、项目风险管控方案 | 风险管理 | 风险识别、管控、应急预案 |
| 十、研究结论及建议 | 总结 | 结论、问题与建议 |
| 十一、附表、附图和附件 | 附件 | 图表、批复文件 |

### 2.2 详细Agent设计

#### Agent 1：需求理解与信息提取Agent
**职责**：理解用户需求，提取项目关键信息

| 功能 | 说明 |
|------|------|
| 意图识别 | 理解"帮我写报告"、"修改报告"等意图 |
| 信息提取 | 项目名称、类型、地点、规模、投资 |
| 需求确认 | 与用户确认信息完整性 |

```
输入：用户对话
输出：结构化的项目信息（ProjectInfo）
```

#### Agent 2：项目概况Agent
**职责**：生成报告第一章"概述"的内容

```
章节：
├── （一）项目概况
│   ├── 项目名称
│   ├── 建设地点
│   ├── 项目单位
│   ├── 建设规模与内容
│   ├── 总投资及资金来源
│   └── 建设工期
├── （二）项目单位概况
├── （三）编制依据
└── （四）主要结论和建议
```

#### Agent 3：项目建设背景与必要性Agent
**职责**：分析项目建设背景和必要性

```
章节：
├── （一）项目建设背景
│   ├── 区域经济社会发展概况
│   ├── 行业现状与发展趋势
│   └── 项目立项背景
├── （二）规划政策符合性
│   ├── 国土空间规划符合性
│   ├── 产业政策符合性
│   └── 重大战略符合性
└── （三）项目建设必要性
    ├── 宏观层面必要性
    ├── 中观层面必要性
    └── 微观层面必要性
```

```
│  │ - 地理位置    │  │ - 气温降水    │        │
│  │ - 地形地貌    │  │ - 灾害性天气  │        │
│  └──────────────┘  └──────────────┘        │
│  ┌──────────────┐  ┌──────────────┐        │

│  │ - GDP规模    │  │ - 地形地貌    │        │

│  │ - 洪涝风险    │  │ - 环境保护区  │        │
```

- 国家统计局
- 气象局
- 地质调查局
- 水利部
- 生态环境部

#### Agent 5：项目建设方案Agent
**职责**：生成技术方案和工程方案

```
章节：
├── （一）技术方案
│   ├── 技术路线选择
│   ├── 核心技术指标
│   └── 工艺流程
├── （二）设备方案
│   ├── 主要设备选型
│   └── 设备清单
├── （三）工程方案
│   ├── 总体布置
│   ├── 主要建筑物
│   └── 结构设计
├── （四）用地用海方案
└── （五）数字化方案
```

**特殊能力**：
- 调用专业数据库
- 工程计算与验算
- 标准规范检索

#### Agent 6：财务分析Agent（用户提出的方向）
**职责**：完成投融资和财务分析章节

```
章节：
├── （一）投资估算
│   ├── 建设投资估算
│   ├── 建设期利息
│   └── 流动资金估算
├── （二）资金筹措
│   ├── 资本金来源
│   └── 债务资金来源
├── （三）盈利能力分析
│   ├── 营业收入估算
│   ├── 成本费用估算
│   ├── 利润估算
│   └── 财务指标计算（IRR、NPV、投资回收期）
├── （四）偿债能力分析
│   ├── 借款偿还期
│   ├── 利息备付率
│   └── 偿债备付率
└── （五）财务可持续性分析
    ├── 现金流量分析
    └── 资金平衡分析
```

**特殊能力**：
- 财务模型计算
- Excel表格生成
- 敏感性分析


**职责**：综合分析，生成结论和建议

```
章节：
├── （一）研究结论
│   ├── 建设必要性结论
│   ├── 技术可行性结论
│   ├── 经济合理性结论
│   ├── 环境可行性结论
│   └── 风险可控性结论
```

#### Agent 10：质量检查Agent (Self-Check)
**职责**：检查生成内容的质量

| 检查项 | 检查内容 |
|--------|----------|
| 完整性 | 是否包含所有必要章节 |
| 一致性 | 数据是否前后一致 |
| 专业性 | 术语、格式是否规范 |
| 可读性 | 表述是否清晰 |
| 合规性 | 是否符合国家标准 |

## 三、工作流程

```
1. 用户输入需求
   ↓
2. 需求理解Agent提取项目信息
   ↓
3. 主控Agent生成任务计划
   ↓
4. 并行执行各专业Agent
   ├── 项目概况Agent
   ├── 所在地概况Agent
   ├── 建设方案Agent
   ├── 财务分析Agent
   ├── 影响分析Agent
   └── 风险分析Agent
   ↓
5. 结论建议Agent整合
   ↓
6. 质量检查Agent审核
   ↓
7. 生成最终报告
   ↓
8. 输出Markdown/PDF
```

## 四、技术实现

### 4.1 框架选择

| 方案 | 适用阶段 | 说明 |
|------|----------|------|
| Next.js API Routes | MVP阶段 | 当前架构，够用 |
| LangGraph | 复杂阶段 | 多Agent编排 |
| FastAPI | 生产阶段 | 高并发需求 |

### 4.2 数据来源

| 数据类型 | 来源 | 获取方式 |
|----------|------|----------|
| 地理气象 | 气象局、统计局 | API调用 |
| 地质条件 | 地质调查局 | 专业数据库 |
| 经济数据 | 统计局 | 公开数据 |
| 财务参数 | 用户输入/默认 | 表单收集 |
