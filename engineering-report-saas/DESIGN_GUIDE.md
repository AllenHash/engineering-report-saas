# ERS系统前端设计指南

## 设计原则

### 1. 禁止使用的元素（降低AI感）
- ❌ Inter / Roboto / Arial 系统字体
- ❌ 白色背景 + 紫色渐变
- ❌ 对称布局 + 均匀分布
- ❌ 纯色背景
- ❌ Emoji图标（使用Lucide图标库）

### 2. 必须思考的设计问题
在设计任何页面时，先回答：
1. 界面解决什么问题？
2. 给谁用？
3. 选什么调性？
4. 什么能让用户记住？

### 3. 字体策略
- 标题：使用有特色的中文字体（如思源黑体、站酷高端黑）
- 正文：使用清晰易读的中文字体（如思源宋体、Noto Sans）
- 代码/数字：使用等宽字体

### 4. 色彩策略
- 主色调：深蓝 (#1e3a5f) + 暖橙 (#f59e0b) 对比
- 强调色：使用锐利的对比色
- 背景：深灰渐变或添加纹理
- 使用CSS变量保持一致性

### 5. 布局策略
- 打破对称，尝试不对称布局
- 使用对角线构图
- 留白要麼极简要麼充足

### 6. 动效策略
- 聚焦高影响力时刻（页面切换、按钮悬停）
- 使用交错延迟创造节奏感
- 避免过度动画

### 7. 资源选择
- 图标：使用 lucide-react 或 iconify
- 图片：使用 Unsplash / Pexels 高质量图库
- 插画：使用 unDraw（可自定义颜色）

### 8. 文案风格
- 具体化："每天节省2小时"而非"提升生产力"
- 口语化，带情绪
- 避免空洞的AI式文案

## 页面设计要求

### 首页（三栏布局）
- 左侧项目列表：深色卡片，微圆角，悬停动效
- 中间对话区：简洁输入框，消息气泡有特色
- 右侧面板：标签页切换流畅，报告预览正式

### 配色方案
```css
--bg-primary: #0f172a;      /* 深蓝黑 */
--bg-secondary: #1e293b;    /* 卡片背景 */
--bg-tertiary: #334155;     /* 输入框 */
--text-primary: #f1f5f9;    /* 主文字 */
--text-secondary: #94a3b8;  /* 次要文字 */
--accent-primary: #f59e0b;  /* 琥珀色强调 */
--accent-secondary: #3b82f6; /* 蓝色辅助 */
--border-color: #475569;    /* 边框 */
```

### 字体配置
```css
font-family: 'Noto Sans SC', 'Source Han Sans CN', sans-serif;
```

## 实现要点
1. 使用CSS变量管理主题
2. 组件使用一致的间距系统（4px网格）
3. 交互元素有明确的状态反馈
4. 保持深色主题一致性
