# 开发任务：关键信息联动功能

## 任务概述
当用户修改项目关键信息时，自动检测并提示需要更新的章节

## 需求背景
用户生成报告后，可能会修改项目基本信息（如投资金额）。需要联动更新相关章节内容。

## 功能需求

### 1. 项目信息编辑面板
- 位置：报告编辑器顶部或侧边栏
- 可编辑字段：
  - 项目名称
  - 建设地点
  - 工程类型
  - 建设规模
  - 估算投资
- 保存后触发联动检测

### 2. 联动影响检测
- 当项目信息变更时，自动分析哪些章节受影响
- 关键词匹配逻辑：
  - 投资金额 → "投资估算"、"财务评价" 章节
  - 建设地点 → "项目选址"、"环境评价" 章节
  - 建设规模 → "工程建设方案"、"投资估算" 章节

### 3. 提示用户
- 弹窗显示受影响章节列表
- 用户选择要更新的章节
- 一键 AI 更新

### 4. AI 自动更新
- 调用 /api/chat 重写相关章节
- 保持专业性和一致性

## 页面位置
- 修改现有：`/editor/[id]/page.tsx`

## 实现建议
1. 在编辑器页面增加项目信息编辑区域
2. 监听信息变更事件
3. 关键词匹配影响章节
4. 弹窗确认后调用 AI 重写

## 验收标准
1. ✅ 可以编辑项目基本信息
2. ✅ 修改信息后显示受影响章节
3. ✅ 可以一键 AI 更新相关章节
4. ✅ 更新后内容正确反映新信息
