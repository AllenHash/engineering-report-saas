# 前端设计规范 - 减少 AI 味

## 核心理念

AI 生成的前端往往"千篇一律"，原因是：缺乏产品思考、平庸的审美、偷懒的实现。

---

## 🚫 禁止使用的设计元素

```
- Inter/Roboto/Arial 等系统默认字体
- 白色背景 + 紫色/蓝色渐变
- 完全对称的网格布局
- 过多 emoji
- 蓝色按钮 + 灰色背景标准组合
- 空占位符（必须用真实图片）
- 纯蓝或纯紫主色调
```

---

## ✅ 三大维度

### 1️⃣ 产品思维 - 深度思考

在写代码前，先明确：
- 核心任务路径：用户完成目标的最短路径
- 边界与异常：断网、空状态、报错如何处理
- MoSCoW 模型：Must-have vs Could-have

### 2️⃣ 审美重构 - 跨界灵感

参考大师设计理念（转译而非模仿）：
- **版式**：非对称分栏、网格秩序+破格、超大标题
- **色彩**：高对比撞色、三原色几何、工业感渐变
- **形态**：曲线切割、体块叠合、模块化卡片、细微纹理
- **动效**：200-300ms 过渡，支持 prefers-reduced-motion

灵感来源：
- 平面设计：Massimo Vignelli, Saul Bass
- 建筑美学：扎哈·哈迪德、隈研吾
- 工业设计：Dieter Rams, 深泽直人, Jony Ive

### 3️⃣ 工程师克制 - 细节

- **图标**：禁止 emoji，使用 Lucide/Heroicons/Tabler
- **图片**：禁止空占位符，使用真实图片（Pexels/unDraw）
- **语义**：正确 HTML5 结构（header/main/section/nav）
- **可访问性**：点击区域 ≥44×44px，对比度符合 WCAG

---

## 📝 开发检查清单

每次前端开发完成后检查：

```
- [ ] 是否有产品思维（不只是堆砌 UI）
- [ ] 是否使用特色字体（非默认）
- [ ] 是否有非对称/破格布局
- [ ] 是否有质感背景（非纯色）
- [ ] 是否使用专业图标（非 emoji）
- [ ] 是否有 200-300ms 动效
- [ ] 是否有真实图片（非占位符）
- [ ] 是否符合可访问性
```
