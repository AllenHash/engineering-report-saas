# 前端设计规范 - 减少 AI 味

## 设计原则

为了避免 AI 生成的前端千篇一律的"AI 味"，开发时需要遵循以下原则：

### 🚫 禁止使用的设计元素

```
- Inter/Roboto/Arial 等系统默认字体
- 白色背景 + 紫色渐变
- 对称的网格布局
- 过多的 emoji
- 蓝色按钮 + 灰色背景的标准组合
```

### ✅ 推荐的设计方向

#### 1. 字体选择
- 使用有特色的字体组合
- 展示字体 + 正文字体搭配
- 避免默认字体

#### 2. 色彩策略
- 主色调 + 锐利强调色
- 避免均匀分布的调色板
- 使用渐变网格、噪点纹理背景
- 深色主题可以更有质感

#### 3. 布局设计
- 尝试不对称布局
- 元素重叠、对角线构图
- 奢侈的留白 或 受控的视觉密度

#### 4. 动效设计
- 聚焦高影响力时刻（页面加载、按钮悬停）
- 使用交错延迟创造节奏感

#### 5. 内容丰富
- 使用真实图片（unDraw, Pexels）
- 使用专业图标（Iconify, Lucide）
- 避免空洞的 AI 文案

---

## 开发检查清单

每次前端开发完成后，检查是否包含AI 味"特征：
以下"- [ ] 使用了系统默认字体
- [ ] 背景是纯色或简单渐变
- [ ] 布局过于对称
- [ ] 缺少真实图片或图标
- [ ] 文案过于空泛
