## 1. 产品定义

### 1.1 产品名称
（暂定）工程可行性报告AI助手

### 1.2 产品定位
通过对话式交互 + 资料上传，帮助工程咨询工程师自动生成各类工程可行性报告。

### 1.3 目标用户
- 大型设计院/咨询公司的咨询工程师
- 中小型事务所
- 个人独立咨询师

### 1.4 支持的工程类型（MVP阶段）
1. 公路工程
2. 市政工程
3. 生态环境工程

## 2. 核心功能

### 2.1 MVP必须功能（P0）
| 功能 | 描述 |
|------|------|
| 用户注册/登录 | 手机号/邮箱注册 |
| 工程类型选择 | 从3种工程类型中选择 |
| 对话式输入 | 用户描述项目需求 |

| 预览+导出 | 网页预览，导出PDF |
| 积分系统 | 购买/消耗/查询余额 |

### 2.2 增强功能（P1）
| 功能 | 描述 |
|------|------|
| 模板选择 | 内置标准模板 |
| 资料上传 | 上传图片/Excel参考 |
| 微信支付 | 集成支付 |

### 2.3 技术优化：文本库方案
为提高AI生成质量，前几章内容采用"文本库+AI"混合模式：

| 章节 | 数据来源 | 可靠性 |
|------|----------|--------|
| 地理位置 | 国家地理信息库 | ✅ 高 |
| 气候条件 | 气象局数据 | ✅ 高 |
| 地形地貌 | 地形图/卫星数据 | ✅ 高 |
| 地质条件 | 地质调查局 | ✅ 高 |
| 水文数据 | 水利部/当地水文站 | ✅ 高 |
| 政策法规 | 国家发改委/地方政府 | ✅ 高 |

## 4. 市场分析

### 4.1 行业背景
- 中国企业级SaaS市场规模约888亿元，年增速13%
- AI技术正在重写所有SaaS
- "先评估、后决策"制度（自1985年起）规定基建项目必须做可行性研究
- 工程咨询行业智能化程度低，存在很大提升空间

### 4.2 市场需求
- 传统方式：人工撰写Word/Excel报告，工作量大
- 代写市场价格：2万+/份
- AI生成价格：199元/份（仅为传统1%）

### 4.3 竞争情况
- 目前无专门针对工程可行性报告的AI SaaS
- 通用AI写作工具（WPS AI等）无法满足工程行业专业需求

## 6. 风险与应对

| 风险 | 应对方案 |
|------|----------|
| AI生成质量不行 | 文本库方案 + 人工校对兜底 |
| 没人买单 | 先用"人工+AI"验证市场需求 |
| 大模型API不稳定 | 备选多个模型（glm/qwen/deepseek） |
| 定价不合适 | 动态调整，小范围测试 |
| 一人忙不过来 | 优先核心功能，自动化能自动的环节 |
| 微信支付接入难 | 手动充值积分过渡 |
